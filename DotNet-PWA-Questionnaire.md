# .NET Core and Progressive Web Application - Questionnaire

**Total Marks: 30**  
**Instructions:**
- Attempt any **three questions (out of five)** from Module 1 (15 marks)
- Attempt any **three questions (out of five)** from Module 2 (15 marks)
- Time: 2 hours

## MODULE 1: Introduction to C#.NET (Attempt any 3 out of 5)

### Q1: .NET Ecosystem Architecture (5 marks)

**a)** Differentiate between .NET Framework and .NET Core/.NET 5+. Explain the advantages of .NET Core over .NET Framework with at least two examples.  
**(3 marks)**

**b)** What is Common Language Runtime (CLR)? Explain its role in executing C# code.  
**(2 marks)**

---

### Q2: C# Language Fundamentals and Control Structures (5 marks)

**a)** Write a C# program that demonstrates the use of different data types (int, string, float, bool, decimal) and perform operations on them.  
**(2 marks)**

**b)** Explain the difference between `if-else`, `switch`, and ternary operator with code examples. Which one would you prefer for checking multiple conditions and why?  
**(3 marks)**

---

### Q3: Object-Oriented Programming Principles (5 marks)

**a)** Define Inheritance and Polymorphism in C#. Provide code examples showing:
   - Single inheritance with method overriding
   - Polymorphic behavior
**(3 marks)**

**b)** What is the purpose of interfaces in C#? How do they differ from abstract classes? Provide a practical example.  
**(2 marks)**

---

### Q4: ASP.NET Core Web Fundamentals (5 marks)

**a)** Explain the ASP.NET Core request lifecycle. Draw or describe the flow from HTTP request to HTTP response, highlighting the role of middleware.  
**(3 marks)**

**b)** Differentiate between MVC and Razor Pages approaches in ASP.NET Core. When would you use each?  
**(2 marks)**

---

### Q5: Exception Handling and REST API Design (5 marks)

**a)** Write a C# code snippet demonstrating:
   - Try-catch-finally block
   - Custom exception handling
   - At least one practical scenario
**(3 marks)**

**(2 marks)**

## MODULE 2: Progressive Web Application (Attempt any 3 out of 5)

### Q1: PWA Concepts and Architecture (5 marks)

**a)** Define Progressive Web Applications (PWA) and explain the three key characteristics: Reliable, Fast, and Engaging.  
**(2 marks)**

**b)** Compare PWAs with Native Apps and Hybrid Apps. Create a comparison table highlighting advantages and disadvantages of each.  
**(3 marks)**

---

### Q2: Service Workers and Caching Strategies (5 marks)

**a)** What is a Service Worker? Explain its lifecycle (Installation, Activation, and Fetch events).  
**(2.5 marks)**

**b)** Explain different caching strategies (Cache First, Network First, Stale-While-Revalidate). Provide code snippets for at least two strategies.  
**(2.5 marks)**

---

### Q3: Web App Manifest and Offline Functionality (5 marks)

**a)** What is a Web App Manifest file? List and explain at least five essential properties (name, short_name, icons, display, start_url, etc.).  
**(2.5 marks)**

**b)** How can a PWA function offline? Explain the role of Service Workers and IndexedDB/LocalStorage in achieving offline functionality with a practical example.  
**(2.5 marks)**

---

### Q4: Responsive Web Design and Security (5 marks)

**a)** Explain the Mobile-First approach in responsive design. How would you implement media queries to support multiple device sizes?  
**(2 marks)**

**b)** Discuss CORS (Cross-Origin Resource Sharing) and JWT (JSON Web Tokens) authentication. How would you implement JWT authentication in an ASP.NET Core + PWA application?  
**(3 marks)**

---

### Q5: PWA Integration and Performance Optimization (5 marks)

**a)** How would you consume an ASP.NET Core Web API from a PWA frontend? Explain the steps involved in making API calls using Fetch API and handling responses.  
**(2.5 marks)**

**b)** What is Lighthouse? Explain how you would use it to test PWA performance and accessibility. Mention at least three metrics it measures.  
**(2.5 marks)**
